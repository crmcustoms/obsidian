# Звіт про міграцію ITCOMMS: Megaplan → Planfix

**Дата завершення:** 21 березня 2026
**Проект:** Перенос даних із CRM Megaplan до Planfix
**Сервер:** Hetzner 65.109.8.78, контейнер itcomms-scripts

---

## Що було перенесено

### 1. Постачальники (fill_supplier)

Заповнено поле "Постачальник" у задачах шаблонів 15 і 7691 на основі ID контрагента з Megaplan.

- Оброблено задач: 2743
- Успішно заповнено: 2042
- Не знайдено відповідності: 701

Для 701 задач, де контрагент у Megaplan не мав аналога в Planfix, виконано додатковий пошук через Megaplan API (fill_supplier_from_megaplan).

---

### 2. Дати оплати (fill_payment_date)

Заповнено поле "Дата оплаты" (поле 138909) у шаблонах 15 і 7691 з фактичної дати оплати рахунку в Megaplan.

- Шаблони: 15 (готівка), 7691 (безготівковий)
- Джерело: поле `actualPaymentDate` рахунку Megaplan

---

### 3. Invoice: дата оплати та податок (fill_invoice_payment)

Заповнено задачі шаблону 21 (Invoice):

- Поле "Invoice Payment Date" (128157) — фактична дата оплати
- Поле "TAX" (128159) — тип оподаткування через datatag (16%, 12%, 0%)

Джерело: рахунки Megaplan, зіставлені за номером рахунку.

---

### 4. Файли (migrate_files)

Перенесено вкладення з угод Megaplan до відповідних задач Planfix. Файли розміщено строго у ті поля, з яких були взяті:

- Файл із поля угоди → у відповідне поле задачі
- Файл із коментаря угоди → у новий коментар задачі
- Файл із опису угоди → у опис задачі

**Маппінг полів:**

| Поле Megaplan (Program 14) | Поле Planfix (шаблон 11) |
|---|---|
| Бриф | 120571 |
| Скан договора | 120579 |
| Скан Приложения / ДС | 120581 |
| Акт выполненных работ | 120585 |

| Поле Megaplan (Program 35) | Поле Planfix (шаблони 15, 7691) |
|---|---|
| Скан оригинала договора | 121019 |
| Скан договора | 120579 |
| Акт скан | 121023 |

**Результати по шаблонах:**

| Шаблон | Назва | Задач оброблено | З угодами Megaplan |
|---|---|---|---|
| 15 | Прочі постачальники Конфети | 1404 | 1097 |
| 7691 | Прочі постачальники безнал | 1880 | 1706 |
| 11 | Текущі клієнти | 739 | 677 |

**Захист від дублів:** реалізовано через SQLite-базу `migrate_files.db`. Кожен файл завантажується до Planfix один раз і кешується. При повторному запуску використовується кеш.

## Технічні деталі

- Усі скрипти зберігаються в репозиторії: github.com/crmcustoms/itcomms_migrations, гілка `claude/review-migration-S7AZD`

- Control API доступний на `http://65.109.8.78:8002`
- Логи зберігаються в контейнері: `<script>.log`
