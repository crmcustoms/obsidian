# GSD — Get Shit Done

→ [[claude-code/README|← Назад к обзору]]

## Описание

Система spec-driven разработки для Claude Code. Решает проблему "context rot" — деградации качества при переполнении контекста. Описываешь что хочешь — система планирует и строит.

Создан для соло-разработчиков без enterprise-театра.

## Установка

```bash
npx get-shit-done-cc --claude --global
```

Устанавливается глобально в `~/.claude/` — работает во всех проектах.

## Обновление

```bash
npx get-shit-done-cc@latest
```

## Основные команды

| Команда | Описание |
|---------|----------|
| `/gsd:help` | Список всех команд |
| `/gsd:new-project` | Новый проект — интервью + план |
| `/gsd:map-codebase` | Анализ существующего кода параллельными агентами |
| `/gsd:build` | Выполнение задач по плану |
| `/gsd:audit-uat` | Аудит и верификация |

## Рекомендуемый запуск

```bash
claude --dangerously-skip-permissions
```

Иначе Claude будет спрашивать разрешение на каждую мелочь.

## Хуки (устанавливаются автоматически)

| Хук | Описание |
|-----|----------|
| `SessionStart` | Проверка обновлений GSD |
| `PostToolUse` | Мониторинг контекстного окна |
| `StatusLine` | Статус в строке интерфейса |
