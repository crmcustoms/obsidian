# Planfix Expert Skill

## Версия
2.0 - Полное руководство по настройке Планфикс

## Описание
Этот скилл содержит полную документацию по настройке Планфикс: от базовых сущностей до сложных автоматизаций. Создан для помощи в внедрении CRM, настройке процессов и обучении пользователей.

---

## 1. БАЗОВЫЕ СУЩНОСТИ ПЛАНФИКС

### 1.1 Объекты (современный подход)

**Объект** — цифровой двойник физического объекта или процесса в Планфикс.

**Примеры объектов:**
- Сделка
- Встреча
- Новость
- Статья базы знаний
- Заявка
- Заказ

**У объекта есть:**
1. **Карточка** — набор полей для описания свойств
2. **Набор статусов** — жизненный цикл объекта
3. **Автоматические сценарии** — реакции на изменения
4. **Кнопки** — автоматизация действий

**Где создавать:** Управление аккаунтом → Объекты

**Важно:** Для новых аккаунтов рекомендуется использовать Объекты вместо старых Шаблонов задач + Процессы.

---

### 1.2 Справочники

**Справочник** — список значений для выбора в полях.

**Типы справочников:**
- Статичный (фиксированный список)
- Динамический (заполняется из других данных)
- Иерархический (с вложенностью)

**Применение:**
- Список городов
- Типы услуг
- Категории клиентов
- Источники лидов

---

### 1.3 Поля (в объектах)

**Типы полей:**

| Тип | Описание | Пример |
|-----|----------|--------|
| Строка | Текст одной строкой | Название компании |
| Текст | Многострочный текст | Описание проекта |
| Число | Целое или дробное | Сумма сделки |
| Дата | Календарная дата | Дата встречи |
| Дата-время | С временем | Срок задачи |
| Список | Выпадающий список | Статус оплаты |
| Флажок | Да/Нет | Важный клиент |
| Контакт | Связь с контактом | Ответственный |
| Ссылка | URL | Сайт клиента |
| Файл | Прикрепление | Договор |

**Вычисляемые поля:**
- Формулы для автоматического расчёта
- Пример: `Сумма * 0.2` (комиссия 20%)

---

### 1.4 Статусы

**Статус** — состояние объекта в процессе работы.

**Стандартный набор для сделки:**
1. Новая
2. Переговоры
3. Заключение договора
4. Ожидание оплаты
5. Выполнена
6. Отменена

**Типы статусов:**
- **Активный** — задача в работе
- **Завершающий** — конечная точка
- **Отменяющий** — отмена/отказ

**Настройка:** Управление аккаунтом → Объекты → [Объект] → Статусы

## 2. АНАЛИТИКА — СЕРДЦЕ ПЛАНФИКС

### 2.1 Что такое аналитика

**Аналитика** — инструмент для добавления реквизитов учёта, не предусмотренных в базовой системе.

**Где используется:**
- В задачах (через скрепку "Прикрепить")
- В проектах
- В отчётах

### 2.2 Стандартные аналитики

| Аналитика | Назначение |
|-----------|------------|
| Управление временем | Учёт рабочего времени |
| Управление финансами | Доходы/расходы |
| Звонки | Журнал звонков из телефонии |

### 2.3 Создание своей аналитики

**Шаги:**
1. Управление аккаунтом → Аналитики → Добавить
2. Указать название (например, "Расходы")
3. Добавить поля:
   - Статья расходов (список)
   - Сумма (число)
   - Дата (дата)
   - Комментарий (текст)
4. Настроить ограничения (в каких статусах доступно)

### 2.4 Итоги аналитик

**Итоги аналитик** — поле для автоматического суммирования данных аналитик.

**Примеры использования:**
- Сумма всех расходов по задаче
- Общее время работы
- Маржинальность по клиенту

**Настройка:**
В шаблоне/объекте → добавить поле типа "Итоги аналитик" → выбрать аналитику и поле для суммирования.

### 2.5 Автоматизация на основе аналитики

**Сценарии на добавление аналитики:**

| Событие | Описание | Пример |
|---------|----------|--------|
| Добавлена любая аналитика | Срабатывает на любую | Общая реакция |
| Добавлена одна строка | Для каждой строки отдельно | Создание подзадачи на каждую строку |
| Добавлена конкретная аналитика | Один раз при добавлении | Формирование счёта |

**Кейс: Создание подзадач из аналитики**
```
Аналитика "Услуги" → 
При добавлении каждой строки →
Создать подзадачу "Оказать услугу: [название]" →
Назначить на [ответственный из поля аналитики]
```

## 3. АВТОМАТИЧЕСКИЕ СЦЕНАРИИ

### 3.1 Что такое сценарий

**Сценарий** — автоматическая последовательность операций при наступлении события.

**Структура:**
1. Событие (триггер)
2. Условия (проверки)
3. Действия (операции)

### 3.2 Типы событий

**Для задач:**
- Создана задача
- Изменён статус
- Изменено поле
- Добавлен комментарий
- Добавлена аналитика
- Изменился исполнитель
- Просрочена задача
- Наступила дата

**Для контактов:**
- Создан контакт
- Изменено поле
- Добавлена связь

### 3.3 Операции в сценариях

| Операция | Описание |
|----------|----------|
| Изменить поле | Установить значение поля |
| Изменить статус | Перевести в другой статус |
| Добавить комментарий | Написать комментарий от робота |
| Создать задачу | Создать подзадачу или связанную |
| Отправить email | Отправить письмо |
| Отправить Post-запрос | Вызвать внешний API |
| Добавить аналитику | Записать данные в аналитику |
| Назначить исполнителя | Поменять ответственного |
| Добавить участника | Добавить в задачу человека |

### 3.4 Условия в сценариях

**Типы условий:**
- Поле равно/не равно значению
- Поле содержит текст
- Поле заполнено/не заполнено
- Исполнитель входит в группу
- Дата наступила

**Логика:**
- ВСЕ условия (AND)
- ХОТЯ БЫ ОДНО (OR)

### 3.5 Примеры сценариев

**Кейс 1: Контроль статуса**
```
Если: Поле "Согласовано" = Да И Поле "Оплачено" = Да
То: Статус = "Выполнено"
```

**Кейс 2: Назначение исполнителя**
```
Событие: Изменено поле "Тип услуги"
Условие: Тип услуги = "Дизайн"
Действие: Назначить исполнителя = "Иванов"
```

**Кейс 3: Мониторинг просроченных сделок**
```
Событие: Наступила дата
Условие: Статус ≠ "Закрыта" И Дата < Сегодня - 15 дней
Действие: 
  1. Отправить email менеджеру
  2. Отправить email руководителю
  3. Если прошло 22 дня → Статус = "В архив"
```

**Кейс 4: Пропущенный звонок**
```
Событие: Добавлена аналитика "Звонок"
Условие: Результат = "Пропущенный"
Действие: Создать задачу "Перезвонить" на ответственного
```

**Кейс 5: Автозаполнение из справочника**
```
Событие: Изменено поле "Город"
Условие: Город заполнен
Действие: Установить поле "Регион" из справочника "Города→Регионы"
```

### 3.6 Особенности работы сценариев

**Важно понимать:**

1. **Порядок выполнения** — сверху вниз в списке
2. **Защита от циклов** — сценарий не срабатывает на свои же изменения
3. **Асинхронность** — работают в очереди, могут быть задержки
4. **Подзадачи** — не чаще 1 раза в 5 минут на одного исполнителя
5. **Права доступа** — при автоматических действиях не проверяются

## 4. ОТЧЁТЫ И АНАЛИТИКА

### 4.1 Типы отчётов

**Стандартные:**
- По задачам
- По проектам
- По сотрудникам
- По контактам
- По аналитикам
- По времени

### 4.2 Настройка отчёта по аналитике

**Структура:**
- Одна строка = одна запись аналитики
- Группировка по реквизитам
- Фильтры по дате, проекту, сотруднику

**Пример: Расходы по месяцам**
```
Тип: Аналитика "Расходы"
Столбцы: Дата, Статья, Сумма
Группировка: Месяц
Фильтр: Дата = Текущий год
```

### 4.3 Планировщик

**Планировщик** — персональный рабочий стол.

**Возможности:**
- Список задач с фильтрами
- Календарь
- Диаграмма Ганта
- Загруженность команды
- Просроченные задачи

**Где настраивать:** Планировщик → Настройка

## 5. ЛУЧШИЕ ПРАКТИКИ НАСТРОЙКИ

### 5.1 Перед началом

1. **Опишите процессы** на бумаге
2. **Определите объекты** (что будете вести)
3. **Нарисуйте воронку** продаж/обработки
4. **Соберите данные** для справочников

### 5.2 Этапы внедрения

**Неделя 1: Фундамент**
- Создать объекты
- Настроить статусы
- Создать справочники
- Добавить базовые поля

**Неделя 2: Формы**
- Настроить карточки объектов
- Создать шаблоны
- Настроить аналитики
- Создать первые задачи

**Неделя 3: Автоматизация**
- Настроить сценарии
- Создать роботов
- Настроить уведомления
- Протестировать процессы

**Неделя 4: Отчёты**
- Создать отчёты
- Настроить планировщики
- Обучить команду
- Запустить в работу

### 5.3 Частые ошибки

❌ **Делать слишком много статусов**
✅ Оптимально: 5-7 статусов

❌ **Создавать слишком сложные сценарии**
✅ Простые цепочки лучше, чем один монстр

❌ **Не тестировать перед запуском**
✅ Создавать тестовые задачи

❌ **Пытаться автоматизировать всё сразу**
✅ Начинать с ручного режима, потом автоматизировать

❌ **Игнорировать обучение команды**
✅ Проводить вебинары, писать инструкции

### 5.4 Рекомендации по структуре

**Проекты:**
- Проект = клиент или направление
- Подпроекты = этапы или подразделения
- Шаблоны проектов для типовых случаев

**Задачи:**
- Использовать объекты для типизации
- Теги для быстрой группировки
- Поля для структурированных данных
- Аналитика для учёта

**Контакты:**
- Сегментация через поля
- История взаимодействий
- Связи между контактами

### 5.5 Интеграции

**Обязательные:**
- Телефония (журнал звонков)
- Email (переписка в задачах)
- Telegram (быстрые уведомления)

**Полезные:**
- Google Calendar (синхронизация)
- 1С (финансы)
- Сайт (заявки)

## 7. API И ИНТЕГРАЦИИ

### 7.1 REST API v2

**Base URL:** `https://[account].planfix.com/rest/`

**Аутентификация:** Bearer Token
```
Authorization: Bearer [token]
```

**Основные endpoint'ы:**
- `POST /task/list` — список задач
- `GET /task/{id}` — задача по ID
- `POST /task` — создать задачу
- `POST /task/{id}` — обновить задачу

### 7.2 XML API v1

**Endpoint:** `https://api.planfix.com/xml/`

**Аутентификация:** Basic Auth (API Key + Token)

**Пример запроса:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<request method="task.getList">
  <account>company</account>
  <pageCurrent>1</pageCurrent>
  <pageSize>50</pageSize>
</request>
```

### 7.3 Вебхуки

**Настройка:**
Управление аккаунтом → Интеграции → Webhooks

**События для подписки:**
- Создание задачи
- Изменение статуса
- Добавление комментария
- Новый контакт
