## FilterTermString — фильтр по строке

```json
{
  "contentType": "FilterTermString",
  "field": "name",
  "comparison": "contains",
  "value": "Иван"
}
```

Варианты `comparison`:

| Значение | Смысл |
|----------|-------|
| `equals` | Точное совпадение |
| `not_equals` | Не равно |
| `contains` | Содержит подстроку |
| `not_contains` | Не содержит |
| `starts_with` | Начинается с |

## FilterTermGroup — логическое объединение

```json
{
  "contentType": "FilterTermGroup",
  "join": "and",
  "terms": [
    {
      "contentType": "FilterTermString",
      "field": "name",
      "comparison": "contains",
      "value": "ООО"
    },
    {
      "contentType": "FilterTermString",
      "field": "status",
      "comparison": "equals",
      "value": "active"
    }
  ]
}
```

`join`: `and` | `or`

## Временный фильтр для Invoice/Contractor

Для сущностей без inline-фильтра — создать фильтр → использовать → удалить.

### 1. Создать фильтр
```
POST /api/v3/invoiceFilter
```
```json
{
  "contentType": "InvoiceFilter",
  "config": {
    "contentType": "FilterConfig",
    "termGroup": {
      "contentType": "FilterTermGroup",
      "join": "and",
      "terms": [{
        "contentType": "FilterTermString",
        "field": "name",
        "comparison": "equals",
        "value": "Счёт №123"
      }]
    }
  }
}
```

### 2. Использовать
```
GET /api/v3/invoice?statusFilterId={filter.id}
```

### 3. Удалить
```
DELETE /api/v3/invoiceFilter/{id}
```

## Фильтры сделок (TradeFilter)

```
GET /api/v3/deal?q=строка поиска&limit=50
```

Или через фильтр по статусу:
```json
{
  "filter": {
    "contentType": "TradeFilter",
    "state": {"contentType": "ProgramState", "id": "10"}
  }
}
```
