## oAuth2 (рекомендуется)

```bash
POST /api/v3/auth/access_token
Content-Type: multipart/form-data

username=user@company.ru
password=secret
grant_type=password
```

Ответ:
```json
{
  "access_token": "YzE5OWEwM...",
  "expires_in": 172800,
  "token_type": "bearer",
  "refresh_token": "YTIzNTM4NW..."
}
```

`expires_in` — 172800 секунд = 48 часов.

---

## Передача токена

### Через заголовок (рекомендуется)
```
Authorization: Bearer YzE5OWEwM...
```

### Через GET-параметр
```
GET /api/v3/deal?{"access_token":"YzE5OWEwM..."}
```

---

## Обновление токена (refresh)

```bash
POST /api/v3/auth/access_token
Content-Type: multipart/form-data

grant_type=refresh_token
refresh_token=YTIzNTM4NW...
```

---

## Cookie

Если пользователь уже залогинен в браузере — передаётся автоматически через куку `COOKIEID1`. Используется для браузерных интеграций.

---

## Текущий пользователь

```
GET /api/v3/currentUser
```

Возвращает [[Сотрудники|Employee]] текущего авторизованного пользователя.
