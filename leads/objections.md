# Ответы на заперечения — Техподдержка

## 1. «ДОРОГО»

**Что думает клиент:**
- Можно найти дешевле
- Погодинка выйдет дешевле
- Это много для «поддержки»

**Ответ:**
```
Давайте порахуємо разом. Скільки зараз витрачається:
- Час власника на технічні питання?
- Очікування виконавців?
- Відволікання менеджерів?
- Втрачені продажі через зриви?

Часто виходить що "дешевше" = дорожче через 
непродуктивність. Фіксована ціна дає прогнозованість.

Давайте спробуємо місяць — якщо не виправдається, 
поверну гроші або розірвемо договір.
```

**Альтернатива:**
- Снижаем первый месяц (скидка 20–30%)
- Пробный период 2 недели
- Начинаем с пакета БАЗОВЫЙ (10К вместо 15К)

## 2. «А ЯКЩО ЗАДАЧ МАЛО?»

**Что думает клиент:**
- Заплачу за воздух
- Часы сгорят
- Лучше по факту платить

**Ответ:**
```
Години не згоряють — накопичуються. 
Невикористані години переходять на наступний місяць 
або витрачаються на покращення системи.

Але чесно скажу: якщо задач дійсно мало — 
можливо вам поки підходить БАЗОВИЙ пакет (10 000₴/міс).

Давайте на діагностиці оцінимо реальний обсяг 
роботи і підберемо оптимальний варіант.
```

**Альтернатива:**
- Переход на пакет БАЗОВЫЙ
- Погодинка по факту (но дороже в час)
- Заморозка контракта на месяц

## 3. «У НАС УЖЕ Є IT-ШНИК»

**Что думает клиент:**
- Зачем дублирование?
- Будут конфликты
- Не хватит задач

**Ответ:**
```
Чудово! Тоді я не заміна — я підсилення.

Ваш IT-шник займається стратегією, 
а я забираю рутину:
- CRM-задачі
- Дрібні доопрацювання
- Налаштування ботів
- Консультації менеджерів

Два сумісних варіанти:
1) Ваш IT подає задачі мені — я вирішую
2) Розділяємо зони: він — стратегія, я — оперативка

Пробували такий формат?
```

**Альтернатива:**
- Консультационный пакет (без исполнения)
- Обучение внутреннего IT
- Совместные звонки по сложным задачам

## 4. «А ЯКЩО ПОТРІБНО ЩОСЬ ВЕЛИКЕ?»

**Что думает клиент:**
- Выйдет за рамки
- Будут скрытые платежи
- Не справитесь

**Ответ:**
```
Все що виходить за рамки пакету — окремий кошторис 
який погоджується ЗАЗДАЛЕГІДЬ.

Правило просте:
- до 3–4 годин = в рамках підтримки
- більше = пишу кошторис, ви погоджуєте

Без сюрпризів. Без авоськи. 
Тільки письмова згода — тоді робота.

Приклад: новий бот = 40 годин = ~40 000₴ окремо.
Погоджуєте — починаю. Ні — шукаєте інше рішення.
```

**Альтернатива:**
- Ежемесячный лимит на крупные задачи
- Сплата частями для больших проектов
- Отдельный договор на разработку

## 5. «ЧОМУ САМЕ ВИ?»

**Что думает клиент:**
- Много других исполнителей
- Чем вы лучше?
- Почему дороже/дешевле?

**Ответ:**
```
Чесно — не всім підходжу. Якщо потрібен 
найдешевший варіант — я не ваш вибір.

Якщо потрібен результат без контролю:
✓ 48+ CRM-проєктів
✓ 2 компанії на підтримці рік+ (можу дати контакти)
✓ Результат видно відразу — не в кінці проєкту
✓ Працюю як партнер, не підрядник

Розумію бізнес процеси, не просто "роблю технічне".
Це відчувається вже на першій розмові.

Давайте 15 хвилин — відчуєте різницю.
```

**Доказательства:**
- Логотипы клиентов (если можно показать)
- Отзывы (текст или аудио)
- Кейсы с цифрами

## 6. «НАМ ЦЕ НЕ ТЕРМІНОВО»

**Что думает клиент:**
- Отложим на потом
- Сейчас не горит
- Подожду лучших времен

**Ответ:**
```
Розумію. Але "не терміново" часто дорого коштує:

→ Задачі накопичуються → вирішувати складніше
→ Конкуренти автоматизуються → ви втрачаєте ринок  
→ Власник втомлюється → рівень рішень падає

Можемо почати з БАЗОВОГО (10 000₴) — 
не великі витрати, але система вже працює.

Або спробуєте так: 2 тижні безкоштовно, 
щоб оцінити користь. Потім вирішите.
```

**Альтернатива:**
- Пробный период
- Отложенный старт (через месяц)
- Консультация без обязательств

## 7. «НАМ ТІЛЬКИ CRM ПОТРІБНА»

**Что думает клиент:**
- Не понимает ценность комплексной поддержки
- Хочет точечное решение
- Боится переплатить

**Ответ:**
```
Зрозуміло. Але CRM без підтримки = дорогий мертвець.

Бачив сотні випадків: впровадили CRM → 
нікому підтримувати → через півроку все в таблицях.

Пропозиція: почнемо з впровадження CRM,  
а перший місяць підтримки — в подарунок.

Так ви отримуєте працюючу систему,  
а не просто "встановлену програму".
```

**Альтернатива:**
- Только внедрение + небольшая переплата за поддержку первый месяц
- Гарантия поддержки в течение месяца
- Обучение сотрудников клиента

## 8. «ДАЙТЕ ДУМАТИ» / «ОБГОВОРЮ З КОМАНДОЮ»

**Что думает клиент:**
- Не уверен
- Нужен повод отказаться
- Откладывает решение

**Ответ:**
```
Звичайно, обміркуйте. Це важливе рішення.

Але досвід показує: "подумаємо" часто перетворюється  
на "забули". Задачі накопичуються, ситуація не змінюється.

Давайте зробимо так: я надішлю короткий план 
на email з ключовими пунктами. Ви покажете команді.

А через 3 дні напишу — може, залишилися питання?  
Без нав'язування, просто допоможу уточнити.

Добре?
```

**Действия:**
1. Отправить email с коммерческим
2. Назначить напоминание через 3 дня
3. Если молчит — мягкий фоллоу-ап
4. Если отказ — оставить на подогрев, писать полезности

---

## Общие принципы ответов

1. **Не спорить** — понять точку зрения
2. **Переводить в цифры** — абстракции vs конкретика
3. **Давать альтернативы** — гибкость вариантов
4. **Брать ответственность** — "давайте попробуем, я поручаюсь"
5. **Создавать срочность** — не агрессивно, но честно о времени
6. **Оставлять дверь открытой** — даже при отказе

---

## Что работает в переговорах

- Короткие примеры из реальной практики
- Цифры: 48 проектов, 2 компании на поддержке, X% роста
- Готовность гарантировать результат или вернуть деньги
- Демонстрация знаний вместо пустых слов
- Уверенность без нахальства
